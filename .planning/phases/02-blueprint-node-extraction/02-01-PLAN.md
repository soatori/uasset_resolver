  <action>
创建 Python 模块 `scripts/node_utils.py`，定义以下函数签名（不实现，仅签名 + docstring + 返回空结构或 None）：

1. `extract_nodes(event_graph: unreal.EdGraph) -> list[dict]`
   - 遍历 event_graph.nodes，构建节点字典列表
   - 调用 extract_single_node() 处理每个节点
   - 返回: `[{'node_class': str, 'node_name': str, 'node_pos_x': int, 'node_pos_y': int, 'node_guid': str, 'pins': list, 'function_reference': dict|None}, ...]`

2. `extract_single_node(node: unreal.EdGraphNode) -> dict | None`
   - 提取单个节点属性：get_class().get_name()、get_fname().to_string()、node_pos_x/y、node_guid
   - 调用 extract_pins() 获取引脚列表
   - K2Node_CallFunction 特有：调用 extract_function_reference()
   - D-12: 异常时返回 None（跳过失败节点）

3. `extract_pins(node: unreal.EdGraphNode) -> list[dict]`
   - 遍历 node.pins，跳过 parent_pin 不为 None 的 SubPin（D-07）
   - 调用 extract_single_pin() 处理每个顶层 Pin
   - 返回引脚字典列表

4. `extract_single_pin(pin: unreal.EdGraphPin) -> dict`
   - 提取 Pin 属性：pin_id、pin_name、direction、pin_type、linked_to、default_value、b_hidden、b_not_connectable、b_advanced_view、b_orphaned_pin
   - 调用 extract_pin_type() 和 extract_linked_to()
   - 处理 SubPins：检查 pin.sub_pins，递归调用提取子 Pin

5. `extract_pin_type(pin_type: unreal.FEdGraphPinType) -> dict`
   - 提取 9 个子字段：pin_category、pin_sub_category、pin_sub_category_object、pin_value_type、container_type、b_is_reference、b_is_const、b_is_weak_pointer、b_is_uobject_wrapper
   - D-06: 完整提取所有子字段

6. `extract_linked_to(pin: unreal.EdGraphPin) -> list[dict]`
   - 遍历 pin.linked_to，转换为 `{node_name: str, pin_id: str}` 格式
   - D-05: LinkedTo 作为 Pin 内嵌属性

7. `extract_function_reference(node: unreal.K2Node_CallFunction) -> dict | None`
   - 提取 FMemberReference：member_name、member_parent、member_guid、b_self_context
   - 仅 K2Node_CallFunction 调用

8. `format_guid(guid: unreal.FGuid) -> str`
   - D-09: 直接使用 UE 返回的原始值（32 字符十六进制）
   - 移除可能存在的连字符，转为大写

每个函数仅定义签名、docstring 和返回占位值（空列表或 None），不实现完整逻辑。
文件头部导入 unreal 模块（类型提示需要）。
添加模块级 docstring 说明用途。
