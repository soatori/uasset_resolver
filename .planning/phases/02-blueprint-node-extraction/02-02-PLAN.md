实现节点提取的两个核心函数：

**extract_nodes(event_graph)**:
- 遍历 `event_graph.nodes`（EdGraph.nodes 属性返回节点数组）
- 对每个节点调用 `extract_single_node(node)`
- D-12: 单个节点提取失败时跳过，记录 warning，继续处理其他节点
- 返回成功提取的节点列表

```python
def extract_nodes(event_graph):
    """提取 EventGraph 的所有节点及其属性（D-01: UE Python API）"""
    nodes = []
    for node in event_graph.nodes:
        node_data = extract_single_node(node)
        if node_data is not None:  # D-12: 跳过失败节点
            nodes.append(node_data)
    return nodes
```

**extract_single_node(node)**:
- 提取节点基本属性：
  - `node_class`: `node.get_class().get_name()` — 获取类名（如 "K2Node_CallFunction"）
  - `node_name`: `node.get_fname().to_string()` — 获取实例名
  - `node_pos_x`: `node.node_pos_x` — 画布坐标 X（snake_case）
  - `node_pos_y`: `node.node_pos_y` — 画布坐标 Y
  - `node_guid`: `format_guid(node.node_guid)` — 32 字符十六进制（D-09）
- 提取引脚列表：调用 `extract_pins(node)`
- 特定节点类型处理：
  - `K2Node_CallFunction`: 调用 `extract_function_reference(node)` 并添加到 `function_reference` 字段
  - `EdGraphNode_Comment`: 提取 `node_comment` 属性（注释框文本）
- D-04: 所有节点类型使用统一字段模型，缺失字段为 None
- 异常处理：try-except 捕获，unreal.log_warning 记录，返回 None

参考 RESEARCH.md Pattern 1 和 Code Examples 第 422-453 行。

实现引脚提取的两个函数：

**extract_pins(node)**:
- 遍历 `node.pins`（UEdGraphNode.pins 属性返回引脚数组）
- D-07: 检查 `pin.parent_pin`，不为 None 则跳过（SubPin 在父 Pin 中处理）
- 对每个顶层 Pin 调用 `extract_single_pin(pin)`
- 返回引脚列表

```python
def extract_pins(node):
    """提取节点的所有引脚（D-07: SubPin 在父 Pin 中处理）"""
    pins = []
    for pin in node.pins:
        if pin.parent_pin is not None:
            continue  # Skip SubPins in main iteration
        pin_data = extract_single_pin(pin)
        pins.append(pin_data)
    return pins
```

**extract_single_pin(pin)**:
- 提取 Pin 属性（D-08: 完整字段提取）：
  - `pin_id`: `format_guid(pin.pin_id)` — FGuid 转 32 字符十六进制
  - `pin_name`: `pin.pin_name.to_string()` — FName 转 string
  - `direction`: `'EGPD_Input'` 或 `'EGPD_Output'`（根据 `pin.direction`）
  - `pin_type`: 调用 `extract_pin_type(pin.pin_type)` — PinType 完整结构
  - `linked_to`: 调用 `extract_linked_to(pin)` — 连线列表
  - `default_value`: `pin.default_value or None` — 默认值字符串
  - `b_hidden`: `pin.b_hidden` — 是否隐藏
  - `b_not_connectable`: `pin.b_not_connectable` — 是否不可连接
  - `b_default_value_is_read_only`: `pin.b_default_value_is_read_only`
  - `b_default_value_is_ignored`: `pin.b_default_value_is_ignored`
  - `b_advanced_view`: `pin.b_advanced_view` — 高级视图引脚
  - `b_orphaned_pin`: `pin.b_orphaned_pin` — 孤立引脚
  - `persistent_guid`: `format_guid(pin.persistent_guid)` 或 None（如存在）
- SubPin 处理：
  - 检查 `hasattr(pin, 'sub_pins')` 和 `len(pin.sub_pins) > 0`
  - 递归提取子 Pin，添加到 `pin_data['sub_pins']` 字段
  - 子 Pin 包含：pin_id、pin_name、parent_pin（父 Pin ID）、linked_to
  - 参考 RESEARCH.md Pattern 2 第 259-277 行

参考 RESEARCH.md Pattern 2 和 Code Examples 第 455-489 行。
