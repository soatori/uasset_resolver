# Plan 04-01: 统一 CLI 入口

**Objective:** 合并 controller.py 和 cue4parse_controller.py 为一个主入口 `scripts/main.py`，提供完整的 CLI 体验：`--help`、`--version`、清晰的参数分组、标准退出码。

## Task 1.1: 创建 scripts/main.py 统一入口

<read_first>
- scripts/controller.py
- scripts/cue4parse_controller.py
- scripts/cue4parse_extractor.py (BPExtractor 接口)
- scripts/formatter.py
</read_first>

<acceptance_criteria>
- scripts/main.py 存在且可执行 (`python scripts/main.py --help` 退出码 0)
- 输出包含：项目名称、版本 "0.1.0"、用法示例、所有参数分组描述
- 支持参数：`uasset`（位置参数）、`--format json|md`、`--backend cue4parse|ue-headless|auto`、`--output PATH`、`--usmap PATH`、`--ue-path PATH`、`--timeout SECS`、`--version`
- 退出码约定：0=成功，1=参数/文件错误，2=提取失败，3=验证失败
</acceptance_criteria>

<action>
创建 scripts/main.py，整合 controller.py 的所有后端逻辑（cue4parse + ue-headless + auto）为一个 argparse CLI。cue4parse_controller.py 保留为独立快速入口（内部调用 main.py）。argparse 使用 `add_argument_group` 分组：基本参数、后端参数、输出参数。
</action>
