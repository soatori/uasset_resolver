# Plan 04-03: 交叉验证 — 输出 vs 参考文件

**Objective:** 创建验证脚本，将工具输出与 `蓝图节点文本参考.md` 中的关键字段进行比对，验证节点数量、节点名称、函数引用、引脚结构的一致性。

## Task 3.1: 创建 scripts/verify.py 验证模块

<read_first>
- 蓝图节点文本参考.md（373 行参考格式）
- scripts/formatter.py (format_md, format_json 输出结构)
- scripts/cue4parse_extractor.py (BPExtractor.extract_to_dict 返回结构)
</read_first>

<acceptance_criteria>
- verify.py 可被 import，提供 verify_output(output_data, reference_path) -> (passed: bool, report: str) 接口
- 验证维度：
  1. 节点数量：输出节点数 >= 参考文件中的节点数（因为 .usmap 缺失可能少提取）
  2. 节点名称：输出中的节点名称集合与参考文件节点名称集合的交集 >= 80%
  3. 函数引用：CallFunction 类型节点的函数名与参考文件匹配
  4. 引脚结构：至少验证引脚 PinName 和 PinCategory 的存在性
- 输出验证报告（stdout），包含每项的 PASS/FAIL 状态
- 退出码 0=全部 PASS，3=部分 FAIL
</acceptance_criteria>

<action>
创建 verify.py 模块。解析 蓝图节点文本参考.md 中的 Begin Object ... End Object 块，提取节点名称列表、函数引用列表、引脚 PinName 列表。然后对比工具输出（dict 格式）的对应字段，生成验证报告。
</action>
