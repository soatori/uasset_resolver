# Milestone v0.1.0: Initial Blueprint Parser

**Status:** ✅ SHIPPED 2026-05-19
**Phases:** 1 - 4
**Total Plans:** 12

## Overview

初始里程碑 — 搭建 Unreal Engine .uasset 蓝图解析工具的基础架构。

目标：从二进制 `.uasset` 文件中提取蓝图事件图（EventGraph）的节点结构（节点类型、函数引用），输出类 UE 编辑器文本格式和结构化 JSON。无需依赖 UE 编辑器运行时即可读取蓝图。

技术路线：
- Phase 1: 验证 UE 无头桥接方案可行
- Phase 2: UE Python API 遇到限制（NodeGuid/Pins/坐标不可访问）→ 切换到 CUE4Parse
- Phase 2B: CUE4Parse C# 二进制解析后端集成成功
- Phase 3: 双格式输出格式化实现
- Phase 4: 统一 CLI + 验证 → 0.1.0 完成

## Phases

### Phase 2: 蓝图节点提取

**Goal**: 从已加载的蓝图资产中提取完整的 EventGraph 结构，包括节点、引脚、连线和画布坐标。
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:

**Details**:
API 限制发现（2026-05-18）：
- NodeGuid = None — UE Python API 不暴露此字段
- NodePosX/Y = 0 — 属性未正确反序列化

替代方案：Phase 2B（CUE4Parse 后端）已启动。

**Goal**: 用 CUE4Parse C# 二进制解析库替代 UE Python API，直接从 .uasset 读取节点数据。
**Depends on**: Phase 1（UE 5.7 环境验证）
**Plans**: 3 plans

Plans:

- [x] 02B-01: Wave 2: Python 封装模块 ✓
- [x] 02B-02: Wave 3: 统一控制器 ✓

**Details**:
成功标准（已调整 — .usmap 暂挂）:

- 后端标识：cue4parse

已知限制:
- .usmap 映射文件缺失 → CUE4Parse 无法解析属性值（坐标/GUID/Pins 为空）
- UE 5.7 无 GenerateMappingFile 命令let
- 潜在解决方案：UE4SS、Dumper-7 注入 UE 编辑器进程生成 .usmap

## Milestone Summary

**Decimal Phases:**
- Phase 2B: CUE4Parse 替代后端（插入在 Phase 2 之后，因 UE Python API 发现限制）

**Key Decisions:**
- Python 控制 UE 无头调用：`.uasset` 二进制复杂，纯 Python 复刻工程量过大 → Phase 1 验证可行，Phase 2 发现 API 限制后切换到 CUE4Parse
- 双格式输出（MD + JSON）：MD 用于对照 UE 编辑器格式验证，JSON 用于程序化消费 → 已实现，两种格式工作正常
- CUE4Parse 替代 UE Python API：UE Python API 不暴露 NodeGuid、NodePosX/Y、Pins → 集成成功，提取速度从 >60s 降到 ~400ms
- .usmap 映射文件问题暂挂：UE 5.7 无内置生成方式 → 下一里程碑探索解决方案

**Issues Resolved:**
- UE Python API 无法访问 NodeGuid/Pins/坐标 → 通过切换到 CUE4Parse 解决架构问题（等待 .usmap 完成数据填充）
- 无统一 CLI → 创建 scripts/main.py 统一入口，支持所有参数
- 输出格式不匹配 UE 编辑器 → formatter.py 正确生成 Begin Object/End Object 块

**Issues Deferred:**
- .usmap 映射文件生成 → 延迟到下一里程碑，需要探索 UE4SS/Dumper-7/UE Python API 导出方案

**Technical Debt Incurred:**
- 坐标/GUID/Pins 字段为空 → 需要 .usmap 才能完整解析，当前版本输出结构正确但缺少数据
- BPExtractor.exe 二进制未提交到 git（~15MB）→ 保持在 engines/bp_extractor/ 被 .gitignore 忽略，用户需要自行编译
